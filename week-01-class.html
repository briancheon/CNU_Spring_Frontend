소개 (Introduction)

웹의 세계에 오신 것을 환영합니다
1 웹(Web)이란 무엇인가?

웹(World Wide Web)은 인터넷 위에서 작동하는 거대한 문서와 서비스의 네트워크입니다. 많은 사람들이 "인터넷"과 "웹"을 혼용하지만, 엄밀히 말하면 다릅니다.

🌐 인터넷 (Internet)

전 세계 컴퓨터들을 연결하는 물리적 네트워크 인프라. 케이블, 라우터, 서버 등 하드웨어와 TCP/IP 같은 프로토콜로 이루어져 있습니다.

🕸️ 웹 (Web)

인터넷 위에서 HTTP/HTTPS 프로토콜을 통해 작동하는 정보와 서비스의 집합. 이메일, FTP 등도 인터넷을 쓰지만 웹은 아닙니다.

💡 비유로 이해하기: 인터넷은 도로망이고, 웹은 그 위를 달리는 자동차들입니다. 도로 위에 자동차만 다니는 게 아니듯, 인터넷 위에 웹만 있는 게 아닙니다.

웹의 간략한 역사

1989
웹의 탄생 팀 버너스리(Tim Berners-Lee)가 CERN에서 연구자들 사이의 정보 공유를 위해 World Wide Web 개념을 제안했습니다. HTML, URL, HTTP — 이 세 가지가 웹의 근본입니다.
1995
JavaScript 등장 넷스케이프(Netscape) 브라우저에서 단 10일 만에 만들어진 언어가 JavaScript입니다. 처음에는 간단한 폼 검증용이었지만, 지금은 웹 개발의 핵심 언어가 되었습니다.
2004
Web 2.0 시대 Gmail, Google Maps 등의 등장으로 웹이 단순 문서에서 인터랙티브 애플리케이션으로 진화했습니다. AJAX 기술이 페이지 새로고침 없이 데이터를 주고받을 수 있게 해줬습니다.
2013
프레임워크 시대 React(2013), Vue(2014), Angular(2016) 등이 등장하면서 대규모 SPA(Single Page Application) 개발이 본격화되었습니다.
2024
AI + 웹 개발 시대 Copilot, Cursor, Claude 등 AI 도구가 개발 프로세스에 깊이 통합되었습니다. 코드 작성뿐 아니라 디자인, 테스트, 배포까지 AI가 보조합니다.

여러분이 브라우저 주소창에 https://github.com을 입력하는 순간, 다음과 같은 일이 일어납니다.

DNS 조회 (Domain Name System) github.com → 140.82.121.4 처럼, 사람이 읽을 수 있는 도메인을 IP 주소로 변환합니다. 전화번호부와 같은 역할입니다.
②
TCP 연결 수립 & TLS 핸드셰이크 서버와 안전한 연결(HTTPS)을 맺습니다. "저 이쪽이에요" "네, 확인했어요, 암호화 시작할게요" 하는 과정입니다.
③

④
서버 응답 서버가 200 OK와 함께 HTML 파일을 전송합니다. HTML 안에 CSS와 JS 파일 링크가 있으면 추가로 요청합니다.
⑤
브라우저 렌더링 받은 HTML을 파싱해 DOM을 만들고, CSS로 스타일을 적용하고, JavaScript를 실행합니다. 우리 눈에 보이는 페이지가 완성됩니다.
2 웹을 구성하는 세 가지 언어

모든 웹 페이지는 세 가지 언어의 조화로 만들어집니다. 이 세 가지의 역할을 명확히 이해하는 것이 웹 개발의 출발점입니다.

HTML (HyperText Markup Language) — 구조 (Structure)
🏗️ 페이지의 뼈대. "이건 제목이야", "이건 문단이야", "이건 이미지야"를 정의합니다. 의미(Semantics)를 담당합니다.
CSS (Cascading Style Sheets) — 스타일 (Style)
🎨 페이지의 외모. 색상, 폰트, 크기, 레이아웃, 애니메이션. "이 제목은 파란색 24px로 보여줘"를 정의합니다.
JavaScript — 동작 (Behavior)
⚡ 페이지의 행동. "버튼 클릭하면 메뉴가 열려", "서버에서 데이터 불러와서 화면에 표시해". 상호작용을 담당합니다.
🍕 피자로 비유하면: HTML은 도우(반죽), CSS는 토핑과 시각적 표현, JavaScript는 배달 주문과 인터랙션입니다. 세 가지 모두 있어야 완성된 피자입니다.
3 프론트엔드란? 백엔드와의 차이

🖥️ 프론트엔드 (Frontend)

사용자가 직접 보고 상호작용하는 부분. 브라우저 위에서 실행됩니다.


  • HTML, CSS, JavaScript
  • React, Vue, Svelte 등 UI 프레임워크
  • 브라우저에서 실행됨
  • UX/UI, 접근성, 성능 최적화
  • 사용자 입력 처리, 화면 렌더링
VS

⚙️ 백엔드 (Backend)

사용자 눈에 보이지 않는 서버 쪽 로직. 서버에서 실행됩니다.


  • Node.js, Python, Java, Go 등
  • 데이터베이스 (PostgreSQL, MongoDB)
  • 서버에서 실행됨
  • 비즈니스 로직, 보안, 데이터 관리
  • API 설계, 인증/인가, 데이터 처리
🌉 풀스택 (Full-Stack)? 프론트엔드 + 백엔드를 모두 다루는 개발자를 풀스택 개발자라고 합니다. 이 강의는 프론트엔드에 집중하되, Supabase를 통해 백엔드의 맛도 경험합니다.

🍽️ 레스토랑 비유

프론트엔드는 레스토랑의 홀(Hall) — 인테리어, 메뉴판, 테이블 배치, 종업원과의 상호작용.
백엔드는 레스토랑의 주방(Kitchen) — 요리 레시피, 식재료 관리, 재고 시스템.
손님(사용자)은 홀만 보지만, 맛있는 음식은 주방 없이 나올 수 없습니다.

4 2026년 프론트엔드 개발자가 된다는 것

웹 개발은 지난 10년간 엄청나게 변했습니다. 2026년의 프론트엔드 개발자는 단순히 "버튼 클릭하면 색 바뀌는 코드" 이상을 합니다.

✅ 지금 실제로 하는 일

  • 복잡한 싱글 페이지 앱(SPA) 개발
  • 다양한 기기에서 작동하는 반응형 UI 구축
  • API 연동 및 실시간 데이터 처리
  • 웹 성능 최적화 (Core Web Vitals)
  • 접근성(a11y) 준수
  • CI/CD 파이프라인 및 자동 배포

🤖 AI가 바꾼 것들 (2024~)

  • GitHub Copilot, Cursor로 코드 자동완성
  • Claude, ChatGPT로 디버깅 및 아이디어
  • v0, Bolt로 UI 프로토타이핑 가속화
  • 그러나 기초 이해 없는 AI 의존은 위험
  • AI를 잘 쓰려면 먼저 직접 짤 줄 알아야 함
💬 솔직한 이야기: AI 도구들이 코드를 상당 부분 생성해줄 수 있지만, "왜" 이렇게 작동하는지 모르면 AI가 틀린 코드를 줘도 모르고 그대로 씁니다. 기초를 배우는 이유가 여기 있습니다.

6 8주 강의 계획

이 강의는 8주 동안 "HTML 한 줄도 모르는 상태"에서 "실제 서비스를 인터넷에 배포하는 상태"까지 데려갑니다.

1주
웹의 작동 방식 + HTML 기초 오늘 클라이언트/서버, HTTP, HTML 문서 구조, 시맨틱 요소, 개발 환경 설정
2주
CSS 기초 + Flexbox + 반응형 디자인 박스 모델, 선택자, 변수, Flexbox 레이아웃, 미디어 쿼리
3주
JavaScript 기초 + DOM 조작 변수, 함수, 배열, 객체, 이벤트 리스너, 동적 UI
4주
비동기 JS + API + Git/GitHub async/await, fetch, REST API 호출, Git 워크플로우
5주
React 기초 — 컴포넌트, Props, JSX Vite 셋업, 컴포넌트 트리, props로 데이터 전달, 리스트 렌더링
6주
React State + Hooks useState, useEffect, 제어 컴포넌트, 상태 끌어올리기
7주
백엔드 — Supabase + (FastAPI 맛보기) PostgreSQL DB, Row Level Security, CRUD, 사용자 인증
8주
배포 + 마무리 프로젝트 발표 🎉 Vercel 배포, 환경변수, README, 실제 URL로 라이브 데모
🎯 최종 목표: 8주 후에는 React로 만든 웹 앱이 Supabase 데이터베이스와 연결되어, 실제 인터넷 URL에서 누구나 접속 가능한 형태로 배포됩니다. 포트폴리오로 쓸 수 있는 수준입니다.
7 AI를 활용하는 법 — 제대로
⚠️ 중요한 전제: AI는 강력한 도구이지만, 기초를 이해하지 못한 채로 AI에 의존하면 오히려 독이 됩니다. AI가 준 코드가 왜 작동하는지 모르면, 고장났을 때 고칠 수 없습니다.

✅ AI를 잘 활용하는 방법

  • 모르는 개념을 대화로 설명 요청하기
  • 내가 짠 코드를 리뷰해달라고 하기
  • 에러 메시지를 그대로 붙여넣고 원인 분석 요청
  • 아이디어 구현 방법을 여러 접근법으로 설명 요청
  • AI가 준 코드를 한 줄씩 이해하고 나서 사용

❌ AI를 잘못 활용하는 방법

  • "전체 코드 짜줘" → 복붙 → 이해 없이 제출
  • AI가 자신있게 말하는 것을 검증 없이 믿기
  • 에러 나면 AI에게 묻고, 또 에러 나면 또 묻고 무한반복
  • 기초 개념을 AI에게 물어보고 직접 연습 안 하기

Claude에게 잘 물어보는 방법

좋은 질문 예시
// ❌ 나쁜 질문
"HTML 짜줘"

// ✅ 좋은 질문
"HTML에서 id와 class의 차이가 뭔가요?
제가 이해하기로는 id는 유일하고 class는 여러 곳에 쓴다고 
생각하는데, 맞나요? 틀렸다면 어떤 부분이 틀렸는지 알려주세요."

// ✅ 더 좋은 질문
"아래 코드에서 에러가 납니다:
[코드 붙여넣기]
에러 메시지: [에러 메시지 붙여넣기]
제가 생각하는 원인은 [추측]인데, 왜 틀렸는지 설명해주세요."

이 강의에서 AI를 활용하는 워크플로우

아래 단계를 꼭 지켜주세요. 순서가 중요합니다.

①
먼저 직접 시도한다 (최소 15분) 코드를 직접 작성해봅니다. 틀려도 괜찮습니다. 시행착오 자체가 학습입니다. 이 15분이 AI에게 물어본 답보다 훨씬 깊은 이해를 줍니다.
②
막히면 에러 메시지를 읽는다 브라우저 DevTools의 Console 탭, VS Code의 에러 표시 등을 먼저 확인합니다. 에러 메시지를 읽는 습관이 개발자의 핵심 역량입니다.
③
그래도 모르겠으면 AI에게 물어본다 "내가 시도한 것 + 에러 내용 + 내 추측"을 함께 보내세요. AI가 정확한 원인을 파악하고 왜 그런지까지 설명해줍니다.
④
AI 답변을 이해한 후 직접 다시 작성한다 복붙 금지! AI가 준 코드를 보고 이해한 후, 에디터를 비우고 기억만으로 다시 작성해봅니다. 이 과정에서 진짜 학습이 일어납니다.

2 HTML이란 무엇인가?

HTML (HyperText Markup Language)은 프로그래밍 언어가 아닙니다. 웹 페이지의 내용과 구조를 정의하는 마크업 언어입니다.

📄 "HyperText"란?

단순한 텍스트를 넘어서, 다른 문서로 이동하는 링크를 포함할 수 있는 텍스트입니다. 웹을 "웹(거미줄)"이라고 부르는 이유입니다.

🏷️ "Markup"이란?

텍스트에 태그(tag)를 붙여서 의미를 부여하는 방식. <h1>제목</h1>처럼 꺾쇠 괄호로 텍스트를 감쌉니다.

태그(Tag)의 구조

HTML 태그 해부
       여는 태그   내용   닫는 태그
          ↓        ↓       ↓
         <h1>안녕하세요</h1>

   속성(attribute)이 있는 경우:
   <a href="https://google.com">구글로 이동</a>
       ↑          ↑
    속성 이름    속성 값

   자기 닫는 태그 (내용 없음):
   <img src="photo.jpg" alt="내 사진" />
   <br />
   <input type="text" />
📌 핵심 규칙: 여는 태그가 있으면 반드시 닫는 태그가 있어야 합니다. 단, <img>, <br>, <input> 등 빈 요소(void element)는 닫는 태그가 없습니다.

시맨틱 HTML (Semantic HTML)

시맨틱(Semantic)이란 "의미를 가진"이라는 뜻입니다. 단순히 화면에 보이는 것 이상으로, 내용의 의미를 명확히 전달하는 태그를 사용해야 합니다.

❌ 비시맨틱 (나쁜 예)

<div>
  <div>내 블로그</div>
  <div>
    <div>최신 글</div>
    <div>내용...</div>
  </div>
</div>

브라우저/검색엔진이 구조를 이해 못 함

→

✅ 시맨틱 (좋은 예)

<body>
  <header>내 블로그</header>
  <main>
    <article>
      <h1>최신 글</h1>
      <p>내용...</p>
    </article>
  </main>
</body>

의미 명확 → SEO + 접근성 향상

3 HTML 문서의 구조

모든 HTML 문서는 정해진 기본 구조를 가집니다. 외워야 할 내용이 아니라 이해해야 할 내용입니다.

최소한의 유효한 HTML 문서
<!DOCTYPE html>          <!-- HTML5 문서임을 선언 -->
<html lang="ko">          <!-- 문서의 언어 (스크린리더, 번역 등에 사용) -->

  <head>                  <!-- 브라우저에게 정보를 주는 영역 (화면에 안 보임) -->
    <meta charset="UTF-8" />        <!-- 한글 등 유니코드 문자 사용 -->
    <meta name="viewport"
          content="width=device-width, initial-scale=1.0" />
        <!-- 모바일 반응형을 위한 설정 -->
    <title>페이지 제목 (탭에 표시됨)</title>
    <link rel="stylesheet" href="styles.css" />
        <!-- 외부 CSS 파일 연결 -->
  </head>

  <body>                  <!-- 실제 화면에 보이는 모든 내용 -->

    <header>             <!-- 페이지 상단 영역 (로고, 네비게이션) -->
      <nav>
        <a href="#about">소개</a>
        <a href="#contact">연락처</a>
      </nav>
    </header>

    <main>               <!-- 페이지의 핵심 내용 (하나만 존재해야 함) -->

      <section id="about">   <!-- 관련 내용의 묶음 -->
        <h1>안녕하세요, 저는 김민준입니다</h1>
        <p>컴퓨터공학과 2학년입니다.</p>
      </section>

      <article>           <!-- 독립적으로 의미있는 내용 (블로그 포스트, 카드 등) -->
        <h2>첫 번째 프로젝트</h2>
        <p>HTML로 만든 프로필 페이지입니다.</p>
      </article>

    </main>

    <footer>             <!-- 페이지 하단 (저작권, 연락처 등) -->
      <p>&copy; 2026 김민준</p>
    </footer>

  </body>
</html>

자주 사용하는 HTML 요소들

텍스트 요소

  • <h1> ~ <h6> — 제목 (h1이 가장 큼)
  • <p> — 문단(단락)
  • <strong> — 굵게 강조
  • <em> — 기울여 강조
  • <span> — 인라인 컨테이너
  • <br> — 줄 바꿈

구조 & 리스트

  • <div> — 블록 컨테이너 (의미 없음)
  • <ul> — 순서 없는 리스트 (●)
  • <ol> — 순서 있는 리스트 (1,2,3)
  • <li> — 리스트 항목
  • <table>, <tr>, <td> — 표

미디어 & 링크

  • <a href="..."> — 하이퍼링크
  • <img src="..." alt="..."> — 이미지
  • <video>, <audio> — 동영상/음성
  • <iframe> — 외부 콘텐츠 삽입

폼 요소

  • <form> — 폼 컨테이너
  • <input type="..."> — 입력 필드
  • <textarea> — 여러 줄 텍스트
  • <button> — 버튼
  • <label for="..."> — 레이블
  • <select>, <option> — 드롭다운
4 DOM 트리 (Document Object Model)

브라우저는 HTML 파일을 읽고 그것을 트리(Tree) 형태의 데이터 구조로 변환합니다. 이것이 DOM입니다. JavaScript로 화면을 조작할 때 실제로 조작하는 것이 바로 이 DOM입니다.

🌳 나무(Tree) 구조: HTML의 중첩 구조는 나무의 가지처럼 부모-자식 관계를 형성합니다. <html>이 뿌리(root)이고, 모든 요소는 그 아래에 위치합니다.

아래 HTML은 브라우저가 DOM 트리로 어떻게 해석하는지 함께 살펴봅니다.

HTML 코드
<html>
  <head>
    <title>내 페이지</title>
  </head>
  <body>
    <h1>안녕!</h1>
    <p>
      클릭하면 
      <a>이동</a>
    </p>
  </body>
</html>
document
└── html
    ├── head
    │   └── title
    │       └── "내 페이지"
    └── body
        ├── h1
        │   └── "안녕!"
        └── p
            ├── "클릭하면 "
            └── a
                └── "이동"

브라우저 DevTools로 DOM 확인하기

실습: Chrome에서 아무 웹사이트나 열고, F12 (또는 마우스 오른쪽 > 검사)를 눌러보세요. Elements 탭에서 DOM 트리를 직접 볼 수 있습니다. 요소를 클릭하면 화면에서 해당 영역이 하이라이트됩니다.

Elements 탭

DOM 구조를 실시간으로 보고 직접 수정 가능

Console 탭

JavaScript 실행, 에러 메시지 확인

Network 탭

서버에 어떤 요청/응답이 오가는지 확인

DOM 용어 정리

DOM 트리에서 자주 나오는 용어들입니다. 3주차(JavaScript)에서 DOM을 직접 다룰 때 다시 등장하니, 지금은 "이런 게 있구나" 정도만 기억하세요.

👨‍👩‍👧 부모(Parent) / 자식(Child)

태그가 다른 태그를 감싸고 있으면, 바깥쪽이 부모, 안쪽이 자식입니다.
<body> 안에 있는 <h1>은 body의 자식이고, body는 h1의 부모입니다.

👫 형제(Sibling)

같은 부모를 공유하는 요소들입니다.
<h1>과 <p>가 둘 다 <body> 안에 있으면, 이 둘은 형제 관계입니다.

🌱 노드(Node)

DOM 트리의 각 항목을 노드라고 합니다. 요소 노드(태그), 텍스트 노드(글자), 속성 노드(attribute) 등 여러 종류가 있습니다.

🏠 루트(Root)

트리의 가장 꼭대기. HTML 문서에서는 <html> 요소가 루트입니다. 실제로 브라우저에서는 document 객체가 최상위입니다.

🎯 DevTools 실습 과제: 여러분의 즐겨찾기 웹사이트를 열고 F12를 누른 후, Elements 탭에서 아무 요소나 더블클릭해서 텍스트를 바꿔보세요. (화면에서만 바뀌고, 실제 서버의 파일은 바뀌지 않습니다!) 이것이 바로 DOM 조작의 시작입니다.

초보자가 자주 하는 실수

❌ 닫는 태그 빠뜨리기

<div>를 열고 </div>로 닫지 않으면 레이아웃이 깨집니다. 태그를 열자마자 닫는 태그를 먼저 작성하는 습관을 들이세요.

❌ 중첩 순서 실수

<p><strong>텍스트</p></strong>처럼 교차 중첩은 금물! 먼저 연 태그를 나중에 닫아야 합니다: <p><strong>텍스트</strong></p>

❌ img에 alt 속성 빼먹기

alt는 이미지의 설명입니다. 시각 장애인용 스크린리더가 이 텍스트를 읽어줍니다. 항상 의미있는 alt 값을 넣으세요.

❌ 모든 것에 <div> 사용

<div>는 의미가 없는 컨테이너입니다. 헤더는 <header>, 네비는 <nav>, 본문은 <main> — 시맨틱 태그를 써주세요.
